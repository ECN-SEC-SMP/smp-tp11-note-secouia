\section{Introduction}

L'objectif de ce TP est la conception du jeu "Aventurier du rail" en C++. 

Nous avons décider d'implémenter 


Les animaux implémentés sont :
\begin{itemize}
  \item \textbf{Lion} : se déplace en diagonale, attaque avec feuille ou ciseaux.
  \item \textbf{Loup} : se téléporte aléatoirement, attaque avec n'importe quel type.
  \item \textbf{Ours} : se déplace dans 8 directions possibles, attaque toujours avec feuille.
  \item \textbf{Pierre} : immobile, attaque toujours avec pierre.
\end{itemize}

Le système d'attaque suit les règles de pierre-feuille-ciseaux : pierre bat ciseaux, ciseaux bat feuille, feuille bat pierre. En cas d'égalité, l'attaquant perd.

\section{Conception des classes}

\subsection{Classe Attaque}

La classe \texttt{Attaque} représente le système d'attaque basé sur pierre-feuille-ciseaux. Elle contient un entier \texttt{type} (0: pierre, 1: feuille, 2: ciseaux) et une méthode \texttt{resoudreAttaque} qui détermine le gagnant.

\begin{minted}{cpp}
class Attaque {
private:
    int type; // 0: pierre, 1: feuille, 2: ciseaux
public:
    Attaque();
    Attaque(int a);
    bool resoudreAttaque(const Attaque &a) const;
    const string getNomAttaque();
};
\end{minted}

\subsection{Classe Animal}

La classe \texttt{Animal} est abstraite et sert de base pour tous les animaux. Elle définit les attributs communs : nom, position (x,y), état vivant, et type d'attaque.

\begin{minted}{cpp}
class Animal {
protected:
    string nom;
    int x, y;
    bool vivant;
    Attaque typeAttaque;
public:
    Animal(int maxX, int maxY);
    Animal(int maxX, int maxY, int a, int b);
    const string getNom();
    const int getX();
    const int getY();
    const bool getVivant();
    const Attaque getAttaque();
    void setVivant(bool v);
    bool attaque(Animal &a);
    virtual void setAttaque(Attaque atq) = 0;
    virtual void setAttaque() = 0;
    virtual void deplace(int maxX, int maxY) = 0;
};
\end{minted}

\subsection{Classes dérivées}

Chaque animal hérite de \texttt{Animal} et implémente ses propres méthodes \texttt{setAttaque} et \texttt{deplace}.

\subsubsection{Lion}

Le Lion choisit aléatoirement entre feuille et ciseaux pour attaquer, et se déplace en diagonale (une case dans chaque direction).

\subsubsection{Loup}

Le Loup peut attaquer avec n'importe quel type et se téléporte à une position aléatoire sur la grille.

\subsubsection{Ours}

L'Ours attaque toujours avec feuille et se déplace dans une des 8 directions avec des pas de 1 ou 2 cases.

\subsubsection{Pierre}

La Pierre attaque toujours avec pierre et ne se déplace pas.

\section{Implémentation}

Le programme principal utilise un \texttt{vector} de \texttt{unique\_ptr<Animal>} pour gérer les animaux de manière polymorphique. La grille est de taille 10x10, avec initialement 25\% des cases occupées par des animaux (alternant Lion, Loup, Ours, Pierre).

La boucle principale effectue les étapes suivantes à chaque tour :
\begin{enumerate}
  \item Déplace chaque animal vivant (sauf les Pierres qui sont immobiles).
  \item Pour chaque animal déplacé, vérifie s'il y a collision avec d'autres animaux sur la même case.
  \item En cas de collision, les animaux concernés choisissent une attaque et combattent selon les règles de pierre-feuille-ciseaux.
  \item Le perdant du combat est marqué comme mort.
  \item Affiche l'état actuel de la grille.
  \item Pause d'une seconde pour observer l'évolution.
  \item Répète jusqu'à ce que seuls les animaux immobiles (Pierres) soient encore vivants.
\end{enumerate}

Les combats résolvent les conflits selon le système pierre-feuille-ciseaux, éliminant les perdants.

\section{Conclusion}

Ce TP nous a permis de mettre en pratique les concepts avancés de la programmation orientée objet en C++ : héritage, polymorphisme, classes abstraites et gestion dynamique de la mémoire avec \texttt{unique\_ptr}. L'ajout du système d'attaque stratégique transforme le jeu de la vie classique en une simulation plus interactive et imprévisible.
